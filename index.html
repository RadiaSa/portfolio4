Periode 1: Computers en Data

In de eerste periode hebben we ons gericht op hoe computers data begrijpen en gebruiken. Dit hebben we in drie hoofdonderdelen verdeeld:

Hoofdstuk 1:

Van Binair naar Hexadecimaal: We begonnen met het basisidee van binaire getallen – dat zijn die 1'en en 0'en waar computers mee werken. Daarna leerden we hoe we deze binaire getallen kunnen omzetten in hexadecimale getallen. Dit is handig omdat hexadecimale getallen makkelijker te lezen zijn voor ons, terwijl computers prima met binaire getallen kunnen werken.

Hoofdstuk 2:

Hoe Computers Bestanden Opslaan: Vervolgens hebben we gekeken naar hoe computers bestanden opslaan. We hebben verschillende soorten bestanden bekeken, zoals tekstbestanden en foto's, en geleerd hoe computers deze informatie bewaren.

Om te verduidelijken hoe computers bestanden opslaan en ophalen, zal ik eerst uitleggen wat er precies gebeurt wanneer een bestand wordt opgeslagen of geopend op een computer. Dit proces bestaat uit verschillende stappen en systemen binnen de computer, die allemaal samenwerken om ervoor te zorgen dat je gegevens veilig en efficiënt worden bewaard en toegankelijk zijn wanneer je ze nodig hebt. Hieronder volgt een gedetailleerde uitleg van dit proces:

Data Representatie: In de basis vertegenwoordigen computers alle data als binaire getallen, een reeks van 1'en en 0'en. Elk karakter, beeld, of geluid wordt omgezet in deze binaire code. Bijvoorbeeld, in tekstbestanden, wordt elk karakter vertegenwoordigd door een unieke binaire code volgens een standaard zoals ASCII of Unicode.

File System: De harde schijf of SSD van een computer is waar bestanden fysiek worden opgeslagen. Deze opslagmedia zijn georganiseerd in een 'file system'. Dit systeem bepaalt hoe data wordt opgeslagen, gevonden en opgeroepen. Bekende bestandssystemen zijn NTFS (Windows), APFS (Apple), en EXT4 (Linux).

Bits en Bytes: Data op een harde schijf wordt opgeslagen in 'bits' en 'bytes'. Een bit is de kleinste data-eenheid en kan een waarde van 0 of 1 hebben. Een byte bestaat uit 8 bits en kan gebruikt worden om een enkel karakter op te slaan, zoals een letter of een cijfer.

Bestandsmeta-data: Naast de eigenlijke data bevat een bestand ook meta-data. Dit omvat informatie zoals de bestandsnaam, de grootte, de datum waarop het laatst gewijzigd is, en waar op de harde schijf het bestand zich bevindt.

I/O Operations: Wanneer een programma een bestand nodig heeft, voert het een 'Input/Output' (I/O) operatie uit. Dit betekent dat het besturingssysteem instructies krijgt om de data van de harde schijf te lezen en beschikbaar te stellen aan het programma.

Buffers en Caching: Om de efficiëntie te verhogen, gebruiken computers 'buffers' en 'caching'. Dit zijn tijdelijke opslaglocaties in het RAM-geheugen van de computer. Als een bestand vaak wordt gebruikt, wordt het tijdelijk in dit snellere geheugen opgeslagen zodat het sneller toegankelijk is.

File Allocation Table (FAT) of Inode Table: Voor de organisatie van bestanden op de harde schijf gebruikt een bestandssysteem een structuur zoals een File Allocation Table (FAT) of een Inode Table. Deze structuren houden bij waar op de harde schijf de stukjes van een bestand zijn opgeslagen, vooral belangrijk als een bestand over meerdere locaties is verspreid.

Schrijf/Leesoperaties: Tot slot, wanneer een bestand wordt opgeslagen ('geschreven') of gelezen, worden er schrijf/leesoperaties uitgevoerd waarbij de data van de harde schijf naar het RAM wordt verplaatst voor verwerking, of andersom.

Periode 2: Programmeren met Python

In de tweede periode hebben we onze programmeervaardigheden ontwikkeld, vooral met de programmeertaal Python. Python is makkelijk te leren en heel handig. Hier zijn een paar dingen die we hebben gedaan:

Functies in Python: We leerden over functies in Python. Dit zijn stukjes code die je steeds opnieuw kunt gebruiken. Het is alsof je een kleine machine maakt die een bepaalde taak doet, en je kunt die machine steeds opnieuw gebruiken in je programma.

Werken met Lijsten: Python heeft iets wat 'lijsten' heet, en dat zijn eigenlijk gewoon een soort lijstjes waar je dingen in kunt opslaan. We hebben geoefend met het maken en gebruiken van deze lijsten.

Programmeren voor Games: Een leuke deel was dat we Python ook hebben gebruikt om dingen te doen in games, met een programma genaamd Unity. Dit was een leuke manier om te zien hoe programmeren kan worden gebruikt in de echte wereld, zoals bij het maken van games.

Unity

Unity is een softwareplatform dat gebruikt wordt voor het ontwikkelen van videogames en andere interactieve toepassingen zoals simulaties en virtual reality ervaringen. Het stelt ontwikkelaars in staat om zowel 2D- als 3D-games te creëren en deze aan te passen met behulp van grafische tools en programmeercode. Unity is populair vanwege de varatie en het vermogen om games te creëren die werken op verschillende platforms, zoals pc's, gameconsoles en mobiele apparaten.

In periode 4 hebben we met Unity gewerkt aan een project om onze eigen game te ontwikkelen. Hoewel we uiteindelijk niet genoeg tijd hadden om de volledige game te bouwen, hebben we wel een concept ontwikkeld. Dit project gaf ons inzicht in de basis van gameontwikkeling en de mogelijkheden die Unity biedt, zelfs als we niet in staat waren het volledige spel af te ronden.
